# 文章分类词提取功能 - Product Requirement Document

## Overview
- **Summary**: 为南京信息工程大学信息公告栏爬虫添加文章分类词提取和标题前缀功能，从目标网页中提取文章标题自带的关键分类词（如"教学考试"、"学术报告"等中文分类标识），并将这些分类词作为前缀附加到对应爬取文件的标题开头。
- **Purpose**: 解决当前爬取的公告文件标题缺乏分类标识的问题，通过提取和添加分类词前缀，提高文件的可读性和分类管理能力。
- **Target Users**: 使用该爬虫的用户，需要对公告进行分类管理和快速查找的用户。

## Goals
- 从目标网页中准确识别和提取文章分类词
- 将分类词作为前缀附加到文件标题开头
- 使用适当的分隔符保持标题可读性
- 确保分类词提取的准确性，避免错误添加或遗漏

## Non-Goals (Out of Scope)
- 不修改文章正文内容的分类信息
- 不添加新的分类词，仅使用网页中已有的分类标识
- 不创建分类目录结构，仅修改文件名前缀
- 不处理无分类标识的文章（保持原标题）

## Background & Context
- 当前项目是南京信息工程大学信息公告栏爬虫
- 爬取的公告保存为Markdown文件，文件名格式为：`YYYY-MM-DD_标题.md`
- 从现有数据看，部分公告标题包含分类词（如"第七周会议日程安排"、"关于做好2026届本科毕业论文..."）
- 需要从列表页或详情页中提取明确的分类标识

## Functional Requirements
- **FR-1**: 从公告列表页或详情页中识别和提取文章分类词
- **FR-2**: 将提取的分类词作为前缀附加到文件标题开头
- **FR-3**: 使用适当的分隔符（如" - "或"【分类】"）连接分类词和原标题
- **FR-4**: 确保分类词与原标题之间保持良好的可读性
- **FR-5**: 处理无分类标识的文章，保持原标题不变

## Non-Functional Requirements
- **NFR-1**: 分类词提取准确率 ≥ 90%
- **NFR-2**: 不显著影响爬取性能（增加时间 < 100ms/篇）
- **NFR-3**: 向后兼容，现有功能不受影响
- **NFR-4**: 代码可读性好，易于维护和扩展

## Constraints
- **Technical**: Python 3.x, BeautifulSoup4, 现有代码结构
- **Business**: 不破坏现有文件格式，确保文件名兼容性
- **Dependencies**: 依赖现有 parser.py 和 storage.py 模块

## Assumptions
- 网页中存在明确的分类标识（如栏目、分类标签等）
- 分类词通常位于标题前或特定的HTML元素中
- 分类词为中文，长度一般为2-8个字符
- 无分类标识的文章占比较小

## Acceptance Criteria

### AC-1: 分类词提取功能
- **Given**: 爬取包含分类标识的公告网页
- **When**: 解析器处理该公告
- **Then**: 能够从网页中提取到正确的分类词
- **Verification**: `programmatic`
- **Notes**: 验证至少3种不同类型的分类词提取

### AC-2: 分类词前缀附加
- **Given**: 已提取到分类词的公告
- **When**: 保存该公告文件
- **Then**: 文件名包含分类词前缀，格式正确
- **Verification**: `programmatic`
- **Notes**: 验证文件名格式为：`YYYY-MM-DD_【分类】原标题.md`

### AC-3: 分隔符可读性
- **Given**: 包含分类词前缀的文件名
- **When**: 用户查看文件名
- **Then**: 分类词与原标题之间有清晰的分隔，易于阅读
- **Verification**: `human-judgment`
- **Notes**: 分隔符应清晰且不影响文件名兼容性

### AC-4: 无分类词处理
- **Given**: 无分类标识的公告网页
- **When**: 解析和保存该公告
- **Then**: 保持原标题不变，不添加错误的分类词
- **Verification**: `programmatic`
- **Notes**: 验证无分类词的文章文件名格式正确

### AC-5: 向后兼容性
- **Given**: 现有爬虫功能
- **When**: 添加分类词功能后运行爬虫
- **Then**: 所有现有功能正常工作，不受影响
- **Verification**: `programmatic`
- **Notes**: 验证爬取、保存、摘要生成等功能正常
