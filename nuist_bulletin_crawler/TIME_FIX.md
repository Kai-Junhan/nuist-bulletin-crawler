# 时间显示一致性修复记录

## 问题描述
系统的announcement功能模块中，时间显示存在不一致问题：
- 标题下方显示的发布时间错误地显示为系统执行爬虫操作的时间
- 正文内容中显示的发布时间是公告的实际发布时间
- 两者应该保持完全一致

## 根本原因分析

1. **列表页日期提取不完善**：`parse_announcement_list` 函数在无法从列表页提取到日期时，会使用 `datetime.now()` 作为默认值
2. **详情页未提取发布时间**：`parse_announcement_detail` 函数没有从详情页中提取和返回准确的发布时间
3. **没有优先使用详情页时间**：`save_announcement` 函数只使用列表页的日期，没有使用详情页中更准确的日期

## 解决方案

### 1. 修改 parser.py - 详情页提取发布时间

**修改点1**：`parse_announcement_list` 函数
- 移除 `from_encoding='utf-8'` 参数（避免警告）
- 当无法提取日期时，返回空字符串 `''` 而不是当前时间

**修改点2**：`parse_announcement_detail` 函数
- 在返回结果中添加 `'publish_date': ''` 字段
- 从 `<span class="arti_update">` 标签中提取发布时间
- 例如：`<span class="arti_update">发布时间：2026-04-15</span>`

### 2. 修改 storage.py - 优先使用详情页时间

**修改点**：`save_announcement` 函数
- 优先使用 `detail.get('publish_date')` 作为最终日期
- 如果详情页没有日期，则使用列表页的日期
- 如果都没有，才使用当前时间作为最后的后备

### 3. 修改 main.py - 更新公告数据

**修改点**：主循环中
- 获取详情后，检查 `detail.get('publish_date')`
- 如果存在，则更新 `ann['date'] = detail['publish_date']`
- 确保生成摘要时也使用准确的发布时间

## 验证结果

### 修复前
- 标题下方：`2026-04-18`（爬虫执行时间）❌
- 正文中：`2026-04-15`（实际发布时间）✅
- 不一致！

### 修复后
- 文件名：`2026-04-15_关于申报2026年度中国气象学会科学技术奖 项目(人选)的公示.md` ✅
- 标题下方：`**发布时间**: 2026-04-15` ✅
- 正文中：`发布时间：2026-04-15` ✅
- 摘要中：`- 发布时间: 2026-04-15` ✅
- 完全一致！

## 多场景测试验证

| 测试场景 | 测试结果 |
|---------|---------|
| 有详情页日期的公告 | ✅ 使用详情页准确时间 |
| 无详情页日期但有列表页日期 | ✅ 使用列表页日期 |
| 两者都没有 | ✅ 使用当前时间作为后备 |
| 重新执行爬虫 | ✅ 时间保持一致，不会改变 |
| 不同发布时间的公告 | ✅ 各自使用自己的发布时间 |

## 技术要点

1. **优先级顺序**：详情页时间 > 列表页时间 > 当前时间
2. **准确来源**：详情页的 `<span class="arti_update">` 是最权威的发布时间
3. **数据一致性**：更新公告对象确保摘要和文件名也使用相同时间
4. **向后兼容**：保留原有的后备机制，不会破坏现有功能
