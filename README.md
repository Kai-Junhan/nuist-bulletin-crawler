# NJ信息工程大学信息公告栏爬虫

## 项目概述

NJ信息工程大学信息公告栏爬虫是一个用于自动化爬取和整理NJ信息工程大学官方信息公告的Python工具。该爬虫能够自动抓取公告列表、提取公告详情、分类标识，并将内容以清晰的Markdown格式保存到本地，方便用户进行查阅、搜索和归档。

### 主要功能特性

- 📋 **自动爬取公告列表**：从首页获取最新公告链接
- 📝 **完整内容提取**：获取公告标题、发布时间、正文内容、附件信息
- 🏷️ **智能分类识别**：自动提取公告分类词并添加到文件名前缀
- 📁 **Markdown格式保存**：清晰的文件结构，便于阅读和编辑
- ⏰ **时间筛选**：可配置只爬取最近N天的公告
- 🔄 **进度显示**：实时进度条，直观了解爬取状态
- 🎮 **交互式菜单**：提供友好的终端交互界面
- 🧹 **数据清理工具**：一键清理已爬取的数据

## 安装指南

### 环境要求

- Python 3.7 或更高版本
- 网络连接（用于访问NJ信息工程大学官网）

### 安装步骤

1. **克隆或下载项目**
   ```bash
   cd nuist_bulletin_crawler
   ```

2. **创建虚拟环境（推荐）**
   ```bash
   python -m venv venv
   
   # Windows
   venv\Scripts\activate
   
   # Linux/Mac
   source venv/bin/activate
   ```

3. **安装依赖**
   ```bash
   pip install -r requirements.txt
   ```

## 使用说明

### 方式一：交互式菜单（推荐）

提供友好的终端交互界面，通过数字选择功能：

```bash
python menu.py
```

菜单选项：
- `0` - 退出系统
- `1` - 配置设置：查看和修改配置参数
- `2` - 启动数据爬取：执行网络数据采集任务
- `3` - 执行文件删除：安全删除data目录中的文件

### 方式二：直接运行主程序

```bash
python main.py
```

### 方式三：使用数据清理工具

如需清除data目录中的所有文件（保留目录结构），运行：

```bash
python clean_data.py
```

数据清理工具功能：
1. 扫描data目录及其所有子目录
2. 显示即将删除的文件数量及路径
3. 要求用户确认后再执行删除
4. 删除所有文件但保留目录结构
5. 处理删除错误并继续执行后续操作
6. 生成详细的操作报告

## 功能特性详解

### 1. 配置设置

通过 `config.py` 可以自定义以下参数：

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `BASE_URL` | 网站基础URL | `https://bulletin.nuist.edu.cn` |
| `LIST_URL` | 公告列表URL | `https://bulletin.nuist.edu.cn` |
| `DAYS_TO_KEEP` | 保留最近N天的公告 | `7` |
| `REQUEST_TIMEOUT` | 请求超时时间（秒） | `30` |
| `REQUEST_RETRIES` | 请求重试次数 | `3` |
| `REQUEST_DELAY` | 请求延迟范围（秒） | `(2, 4)` |

### 2. 分类词提取

爬虫能够智能识别公告的分类信息，包括：
- 教学考试
- 科研信息
- 党政事务
- 招标信息
- 会议通知
- 招生就业
- 创新创业
- 学术研讨
- 专题讲座
- 校园活动
- 学院动态
- 文件公告
- 学术报告
- 其他

**文件命名格式**：`YYYY-MM-DD_【分类词】公告标题.md`

### 3. 数据存储

爬取的公告数据以 Markdown 格式保存，包含：
- 公告标题
- 发布时间
- 原始链接
- 正文内容
- 附件列表

### 4. 进度显示

使用 `tqdm` 库提供实时进度条，显示：
- 爬取进度百分比
- 已处理/总数量
- 已用时间/预估剩余时间
- 处理速度
- 当前处理的公告标题

## 常见问题

### Q: 如何修改要爬取的天数？

A: 编辑 `config.py` 文件，修改 `DAYS_TO_KEEP` 参数的值。

### Q: 爬取速度太慢怎么办？

A: 可以调整 `config.py` 中的 `REQUEST_DELAY` 参数，减小延迟范围。但请注意，过短的延迟可能会触发反爬机制。

### Q: 如何只爬取特定分类的公告？

A: 当前版本不支持分类筛选，会爬取所有分类的公告。可以在保存后通过文件名中的分类词进行筛选。

### Q: 遇到网络错误怎么办？

A: 程序内置了重试机制，会自动重试 `REQUEST_RETRIES` 次。如果仍然失败，请检查网络连接。

### Q: 如何重新开始爬取？

A: 运行 `python clean_data.py` 清理已有的数据，然后重新运行爬虫即可。
